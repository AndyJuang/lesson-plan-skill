# 教案生成 Workflow — 5 階段協議

## Phase 1 — Intake（確認輸入）

**目標**：在動手之前，把來源資料與需求盤點清楚，缺的問、齊的鎖。

**動作清單**：

1. 列出使用者已提供的來源資料：檔名、格式、摘要（若是 PDF，先用 `Read` 快速抓首頁；若是單字表，直接列出）。
2. 列出使用者已提供的授課需求，對照以下檢核表：
   - [ ] 學科 / 單元主題
   - [ ] 學生年級 / 程度
   - [ ] 節數 / 時長
   - [ ] 文化錨點（可選，但若有則必須明確）
   - [ ] 特殊要求（可選）
3. 若有任何必填欄位缺漏，**一次性列出所有問題問使用者**（不要一問一答）。
4. 使用者補齊後，回報鎖定的輸入摘要，徵求「開始生成」確認（可簡短：「輸入已確認，開始生成？」）。

**輸出**：一段「輸入摘要」回報，包含來源清單與需求表。

---

## Phase 2 — Extract（抽取骨架）

**目標**：從來源資料抽出可作為後續生成基石的「單元骨架」。

**動作清單**：

1. 若來源為 PDF / 文件，用 `Read` 讀入，抽出：
   - 核心單字（列表：單字 / 詞性 / 中文意思 / 例句）
   - 核心句型（列表：句型 / 用途 / 例句）
   - 單元學習目標（認知、技能、情意各一至兩項）
2. 若來源為單字表或純文字，直接整理為上述三類結構。
3. 若來源不足以支撐單元教學，**提醒使用者補充資料**，不要硬生出內容。
4. 產出一份「單元骨架」暫存於工作記憶，不寫入輸出資料夾（純內部使用）。

**輸出**：一段骨架摘要回報（單字數、句型數、學習目標），並進入 Phase 3。

---

## Phase 3 — Generate（生成檔案）

**目標**：依 `templates/` 中的 6 份範本，產出對應檔案到輸出資料夾。

**動作清單**：

1. 建立資料夾 `./教案輸出/<主題>_<YYYYMMDD>/`。
2. **可平行產出**的檔案（互不依賴）：
   - `01_教師手冊.md`（依 `templates/01_teacher_notes.md`）
   - `03_學習單.md`（依 `templates/03_worksheet.md`）
   - `04_測驗題目.md`（依 `templates/04_quiz.md`）
   - `05_圖像提示詞.md`（依 `templates/05_image_prompts.md`）
3. **需依賴前項成果**的檔案（依序產出）：
   - `06_雙語對話腳本.md`（依 `templates/06_dialogue_script.md`，僅語言科產出）
   - `02_簡報大綱.md`（依 `templates/02_slides_outline.md`，整合所有前述素材作為簡報骨架）
4. 每份檔案開頭加上 YAML frontmatter：
   ```yaml
   ---
   主題: {{單元主題}}
   對象: {{年級 / 程度}}
   時長: {{節數 / 分鐘}}
   文化錨點: {{如有}}
   生成日期: {{YYYY-MM-DD}}
   ---
   ```

**輸出**：每檔案完成時送一則短回報（檔名 + 關鍵數據，如「05_圖像提示詞.md — 12 組」）。

---

## Phase 4 — Lint（繁中潤稿）

**目標**：所有中文輸出必須通過 zhtw-mcp 校驗。

**動作清單**：

1. 對每份 `.md` 檔（含 `INDEX.md`）執行：
   ```
   mcp__zhtw-mcp__zhtw
     text: {檔案內容}
     content_type: markdown
     fix_mode: lexical_contextual
     fix_output: search_replace
     output: compact
   ```
2. 若回傳有 `applied_fixes`，用 `Edit` 把 search/replace 套回檔案。
3. 若 `issues` 為 warning 且涉及專有名詞（Nanobanana、Atayal、CEFR、Gaga 等），略過不改；若涉及真正錯字，套用修正。
4. 圖像提示詞檔（`05_圖像提示詞.md`）內容為英文，不需跑 zhtw-mcp；僅潤其中文註解。

**輸出**：潤稿報告（幾份檔案、共修幾處）。

---

## Phase 5 — Package（打包交付）

**目標**：產出 `INDEX.md`，彙整檔案清單與使用說明，並回報資料夾路徑。

**動作清單**：

1. 寫 `INDEX.md`：
   - 教案包基本資料（主題、對象、時長、文化錨點）
   - 檔案清單（連結、一行說明、字數或頁數）
   - 使用流程（建議操作順序：印學習單 → 產圖 → 做簡報 → 上課 → 發測驗）
   - 使用前檢核（族語拼寫、文化典故待確認項目）
   - 建議審核對象（族語老師、文健站、校本課程委員⋯⋯）
2. 對 `INDEX.md` 跑 zhtw-mcp 潤稿。
3. 回報最終總結（依 `SKILL.md` 的「回報格式」區塊）。

**輸出**：完整交付報告，包含資料夾路徑與待確認項目清單。

---

## 錯誤處理

- 若 Phase 2 發現來源資料不足（如單字表只有 2 個字），**停下來請使用者補充**，不要硬生。
- 若 zhtw-mcp 不可用，改用人工檢查並在最終回報中標注「未經 MCP 潤稿」。
- 若使用者中途要求修改某一份檔案，直接重跑該檔案的 Phase 3 + Phase 4，不用重跑全部。

## 效率要點

- Phase 3 中的 4 份平行檔案，應使用「一次訊息多 Write」的方式並行產出。
- 每階段回報**不超過 30 字**，避免打斷使用者工作節奏。
- 全流程目標：10 分鐘內完成一份完整教案包。
